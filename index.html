üì• Installing Claude Code in Replit

Open the Shell tab in Replit and run:

üì• Install & Start Claude Code

npm install -g @anthropic-ai/claude-code && claude
‚ö†Ô∏è Note: Replit's shell resets between sessions, so you'll need to run this install command every time you start a new Replit session.

üÜï First-Time Setup: Create CLAUDE.md

‚ö†Ô∏è Do this once when starting a new project. This creates the CLAUDE.md file with all the safety rules built in.

üìÑ CLAUDE.md Template: Click to Expand üëá

Copy This Into Claude

Create a CLAUDE.md file in the project root with this content: # Project: [Project Name] ## Rules for Claude (Always Follow) ### CLAUDE.md Management - Always read this file at the start of every session. - Update the Session Log at the end of every session with completed tasks and fixes. ### GitHub Safety - Always commit the current working code before starting edits. - If a change may break existing functionality, create a new branch and explain why. - Write clear, descriptive commit messages for all changes. ### Preserve Functionality - Do not remove or alter working features unless explicitly instructed. - If a change risks regression, stop and explain the impact before proceeding. ### Code Changes - Keep changes incremental and well-commented. - Highlight what was added, removed, or updated. ### Verification - After edits, run tests or basic checks to confirm nothing is broken. - Report any errors, warnings, or issues that appear. ### General - Treat GitHub as the single source of truth. - All steps must be safe, reversible, and transparent. --- ## Project Overview [Brief description of what this project does] ## Tech Stack - [List technologies, frameworks, dependencies] ## Project Structure [Key folders and files] --- ## Current Status - Status: [In Progress / Stable / Needs Review] - Last Session: [Date] - Current Branch: main ## Pending Tasks - [ ] [Task 1] - [ ] [Task 2] ## Known Issues - [None yet] --- ## Session Log ### [Date] - Initial Setup - Tasks Completed: - Created CLAUDE.md - [Other setup tasks] - Fixes Applied: - None - Notes: - Project initialized --- ## Project-Specific Rules [Add any coding conventions or requirements specific to this project]

üìå After Claude Creates the File: Commit It

git add CLAUDE.md git commit -m "docs: add CLAUDE.md with project rules and session tracking" git push

‚òÄÔ∏è Morning Kick-Off Prompt

Use this at the start of every coding session.

‚òÄÔ∏è Start of Session

I'm starting a new coding session. Please: 1. Read CLAUDE.md for project context, rules, and pending tasks. 2. Run `git pull` to ensure we're up-to-date. 3. Run `git status` and `git log --oneline -5` to see recent activity. 4. Summarize where we left off and what's pending. 5. Suggest the best next step.

üåô End-of-Day Wrap-Up Prompt

Use this at the end of every coding session.

üåô End of Session

I'm finishing for the day. Please: 1. Confirm all changes are committed to GitHub. 2. If anything is in progress, save it on a separate branch. 3. Update CLAUDE.md: - Add today's date to Session Log - List tasks completed - List fixes applied - Add any notes for next session - Update Pending Tasks - Add any new Known Issues 4. Commit CLAUDE.md: "docs: update CLAUDE.md session log [date]" 5. Run `git push` 6. Confirm the project is in a safe state.

üìä Quick Reference

When What to Do
Every Session Install Claude Code and start it
New Project Run the First-Time Setup to create CLAUDE.md
Start of Session Use the Kick-Off Prompt
End of Session Use the Wrap-Up Prompt
During Session Just work. Rules are already in CLAUDE.md

‚ö° TL;DR

  1. Every session: npm install -g @anthropic-ai/claude-code && claude
  2. New project: Create CLAUDE.md (use First-Time Setup above)
  3. Daily workflow: Kick-off prompt ‚Üí Work ‚Üí Wrap-up prompt
  4. Rules live in CLAUDE.md. No need to repeat them!
